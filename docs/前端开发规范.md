# 前端界面开发规范

> 适用于 `ruoyi-vue3-antd` 项目，基于 Ant Design Vue 4 + BearJiaProTable。

---

## 一、页面结构

每个 CRUD 页面遵循统一的三层结构：

```
页面.vue
├── ProTable（搜索 + 工具栏 + 表格 + 分页）
├── Modal（新增/编辑弹窗）
└── Drawer 或 Modal（详情查看）
```

### 推荐模板骨架

```vue
<template>
  <div>
    <!-- 1. ProTable：自动处理搜索、分页、加载、行选择 -->
    <ProTable
      ref="proTableRef"
      :api="tableApi"
      :columns="columns"
      :searchFields="searchFields"
      :initialSearchParams="initialSearchParams"
      :exportConfig="exportConfig"
      rowKey="id"
    >
      <!-- 操作按钮插槽 -->
      <template #actions="{ selectedRowKeys, selectedRows }">
        <a-button type="primary" @click="handleAdd" v-hasPermi="['module:entity:add']">
          新增
        </a-button>
        <a-button
          :disabled="selectedRowKeys.length !== 1"
          @click="handleEdit(selectedRows[0])"
          v-hasPermi="['module:entity:edit']"
        >
          修改
        </a-button>
        <a-button
          danger
          :disabled="!selectedRowKeys.length"
          @click="handleDelete(selectedRowKeys)"
          v-hasPermi="['module:entity:remove']"
        >
          删除
        </a-button>
        <a-button @click="handleExport" v-hasPermi="['module:entity:export']">
          导出
        </a-button>
      </template>
    </ProTable>

    <!-- 2. 新增/编辑弹窗 -->
    <a-modal
      v-model:open="open"
      :title="title"
      width="800px"
      @ok="submitForm"
      @cancel="open = false"
      :confirmLoading="submitting"
    >
      <a-form ref="formRef" :model="form" :rules="rules" :label-col="{ span: 6 }" :wrapper-col="{ span: 16 }">
        <a-row :gutter="16">
          <a-col :span="12">
            <a-form-item label="字段名" name="field">
              <a-input v-model:value="form.field" placeholder="请输入" />
            </a-form-item>
          </a-col>
          <!-- 更多字段... -->
        </a-row>
      </a-form>
    </a-modal>

    <!-- 3. 详情抽屉（可选） -->
    <a-drawer v-model:open="detailOpen" title="详情" width="760px">
      <a-descriptions bordered :column="1">
        <a-descriptions-item label="字段1">{{ detail.field1 ?? '-' }}</a-descriptions-item>
      </a-descriptions>
    </a-drawer>
  </div>
</template>
```

## 二、ProTable 使用规范

### 必传属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `api` | `{ list, delete? }` | **必须**。`list` 返回 `{ rows, total }` 格式 |
| `columns` | `Array` | **必须**。Ant Design 表格列配置 |
| `rowKey` | `String` | **必须**。主键字段名，如 `'id'` |
| `searchFields` | `Array` | 搜索字段配置，空数组则隐藏搜索 |
| `initialSearchParams` | `Object` | 初始搜索参数 |
| `exportConfig` | `{ url, fileName }` | 导出配置 |

### API 格式要求

```js
// API 返回格式
list(params) → { data: { rows: [...], total: 100 } }

// ProTable 的 api 配置
const tableApi = {
  list: listXxx,        // 列表查询
  delete: delXxx,       // 批量删除（可选，有的话工具栏自动处理）
}
```

### 搜索字段配置

```js
const searchFields = [
  { name: 'code', label: '编号', type: 'input' },
  { name: 'name', label: '名称', type: 'input' },
  { name: 'status', label: '状态', type: 'select', options: [...] },
  { name: 'categoryId', label: '分类', type: 'input', hidden: true }, // 隐藏字段供程序设值
]
```

### 暴露方法（通过 ref 调用）

```js
const proTableRef = ref()
proTableRef.value.refresh()       // 刷新表格
proTableRef.value.searchFormData  // 获取当前搜索参数
```

## 三、权限控制规范

### 指令：`v-hasPermi`

```vue
<!-- 单个权限 -->
<a-button v-hasPermi="['mes:purchaseOrder:add']">新增</a-button>

<!-- 多个权限（满足任一即可） -->
<a-button v-hasPermi="['admin', 'mes:purchaseOrder:add']">新增</a-button>
```

### 权限命名规范

```
模块前缀:实体:操作

mes:purchaseOrder:add      # MES - 采购订单 - 新增
mes:purchaseOrder:edit     # MES - 采购订单 - 编辑
mes:purchaseOrder:remove   # MES - 采购订单 - 删除
mes:purchaseOrder:query    # MES - 采购订单 - 详情
mes:purchaseOrder:approve  # MES - 采购订单 - 审核
mes:purchaseOrder:unapprove # MES - 采购订单 - 弃审

mes:purchaseReceipt:inspect  # MES - 到货单 - 质检
mes:purchaseReceipt:reference # MES - 到货单 - 参照
```

### 所有操作按钮必须加权限

| 按钮 | 权限后缀 | 示例 |
|------|----------|------|
| 新增 | `:add` | `mes:purchaseOrder:add` |
| 编辑 | `:edit` | `mes:purchaseOrder:edit` |
| 删除 | `:remove` | `mes:purchaseOrder:remove` |
| 详情 | `:query` | `mes:purchaseOrder:query` |
| 审核 | `:approve` | `mes:purchaseOrder:approve` |
| 弃审 | `:unapprove` | `mes:purchaseOrder:unapprove` |
| 导出 | `:export` | `mes:material:export` |
| 导入 | `:import` | `mes:material:import` |
| 特殊操作 | 自定义 | `mes:purchaseReceipt:inspect` |

## 四、API 层规范

### 目录结构

```
src/api/
└── mes/
    ├── base/
    │   ├── material.js        # 物料 API
    │   └── materialCategory.js
    └── purchase/
        ├── order.js           # 采购订单 API
        ├── receipt.js         # 到货单 API
        ├── inbound.js         # 入库单 API
        └── inventory.js       # 库存余额 API
```

### API 文件模板

```js
import request from '@/utils/request';

export const xxxApi = {
  list:   (q)     => request({ url: '/mes/xxx/list',    method: 'get',  params: q }),
  get:    (id)    => request({ url: `/mes/xxx/${id}`,   method: 'get' }),
  add:    (data)  => request({ url: '/mes/xxx',         method: 'post', data }),
  update: (data)  => request({ url: '/mes/xxx',         method: 'put',  data }),
  remove: (ids)   => request({ url: `/mes/xxx/${ids}`,  method: 'delete' }),
};

// 特殊操作的 API
export const approveXxx = (id) =>
  request({ url: `/mes/xxx/${id}/approve`, method: 'post' });
```

### 导入规范

```js
// ✅ 推荐：从独立模块导入
import { xxxApi, approveXxx } from '@/api/mes/xxx'

// ⚠️ 可用但不推荐：从统一导出导入
import { xxxApi } from '@/api/mes/xxx.js'
```

## 七、按钮状态联动

```vue
<!-- 新增：始终可用 -->
<a-button type="primary" @click="handleAdd" v-hasPermi="[...]">新增</a-button>

<!-- 编辑：选中一行才可用 -->
<a-button
  :disabled="selectedRowKeys.length !== 1"
  @click="handleEdit(selectedRows[0])"
  v-hasPermi="[...]"
>修改</a-button>

<!-- 删除：至少选中一行才可用 -->
<a-button
  danger
  :disabled="!selectedRowKeys.length"
  @click="handleDelete(selectedRowKeys)"
  v-hasPermi="[...]"
>删除</a-button>

<!-- 状态驱动的操作（表格内） -->
<a v-if="record.status === 'DRAFT'" v-hasPermi="[...]" @click="approve(record)">审核</a>
<a v-if="record.status === 'APPROVED'" v-hasPermi="[...]" @click="unapprove(record)">弃审</a>
```

## 八、文件命名规范

```
views/mes/
├── purchase/
│   ├── order.vue          # 每个实体独立一个页面文件
│   ├── receipt.vue
│   ├── inbound.vue
│   └── inventory.vue
├── base/
│   ├── material/
│   │   └── index.vue      # 复杂页面可用文件夹
│   └── materialCategory/
│       └── index.vue
```

**原则：一个实体一个 `.vue` 文件，禁止多个实体共用一个页面通过 `type` 切换。**

## 十、对比：ProTable vs 裸 a-table

| 功能 | ProTable | 裸 a-table |
|------|----------|-----------|
| 搜索表单 | 配置 `searchFields` 即可 | 手写 a-input + 事件 |
| 分页 | 自动处理 | 手写 pagination + pageChange |
| 加载状态 | 自动处理 | 手写 loading ref |
| 行选择 | 自动处理 | 手写 rowSelection |
| 刷新按钮 | 内置 | 手写 |
| 导出 | 配置 `exportConfig` | 手写 |
| 列设置 | 内置 | 不支持 |
| 列宽调整 | 内置 | 不支持 |
| 异常重试 | 内置 | 不支持 |
| 工具栏 | 内置 | 手写 |

**结论：有 ProTable 就用 ProTable，不要裸写 a-table。**
